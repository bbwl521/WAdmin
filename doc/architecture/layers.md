# 分层设计

## 为什么需要分层？

分层架构将应用程序按职责分离，每层专注于特定的功能，降低耦合度，提高代码可维护性和可测试性。

## 各层职责

### Controller 层

| 允许 | 禁止 |
|------|------|
| 接收请求参数 | 编写业务逻辑 |
| 请求参数校验 | 直接操作 Model |
| 调用 Service 方法 | 跨层调用 Repository |
| 构建响应数据 | 事务管理 |

### Service 层

| 允许 | 禁止 |
|------|------|
| 实现业务逻辑 | 处理 HTTP 请求 |
| 调用 Repository | 直接访问请求对象 |
| 事务管理 | 返回 HTTP 响应 |
| 事件调度 | - |

### Repository 层

| 允许 | 禁止 |
|------|------|
| 数据查询封装 | 包含业务逻辑 |
| 复杂条件构建 | 处理验证逻辑 |
| 分页处理 | - |

### Model 层

| 允许 | 禁止 |
|------|------|
| 定义表映射 | 编写查询逻辑 |
| 定义关联关系 | 包含业务逻辑 |
| 属性转换 | - |
