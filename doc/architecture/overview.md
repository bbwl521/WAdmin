# 架构概览

WAdmin 采用经典的分层架构模式，基于 Hyperf 3.1 协程框架构建。

## 架构图

```
┌─────────────────────────────────────────────┐
│               HTTP / CLI 入口                │
├─────────────────────────────────────────────┤
│              Middleware (中间件)              │
│         Auth / CORS / Log / Validation       │
├─────────────────────────────────────────────┤
│            Controller (控制器层)              │
│           参数校验 · 调用 Service             │
├─────────────────────────────────────────────┤
│             Service (服务层)                  │
│         业务逻辑 · 事务 · 事件                 │
├─────────────────────────────────────────────┤
│           Repository (仓库层)                 │
│           数据查询 · 封装数据访问              │
├─────────────────────────────────────────────┤
│             Model (模型层)                    │
│         ORM映射 · 关联 · 类型转换              │
├─────────────────────────────────────────────┤
│           MySQL / Redis / 文件存储            │
└─────────────────────────────────────────────┘
```

## 请求生命周期

1. 请求进入 Hyperf HTTP Server（协程处理）
2. 经过全局中间件（CORS、日志等）
3. 匹配路由，执行路由中间件（认证、权限）
4. 控制器接收请求，进行参数验证
5. 调用 Service 层执行业务逻辑
6. Repository 层完成数据持久化
7. 返回 JSON 响应给客户端

## 协程优势

Hyperf 基于 Swoole 协程，所有请求在协程中并发处理：

- 数据库连接池复用
- Redis 连接池复用
- 非阻塞 I/O
- 常驻内存，避免重复加载
